# 2\.云保存功能的添加

在上传歌词与翻译的过程中，网页容易因误点而导致上传内容消失，也有因网络问题导致上传失败而使劳动成果丢失的情况，除此之外，在翻译较为长篇的歌曲时，往往会分几次翻译完（今天翻译十句，明天再翻译十句之类的），建议增加系统实时保存功能，实现随写随存，确保编辑内容的安全性。

这点可以参考一下音乐专栏那里，我从社群里的小编那里得到一些反馈，保存功能有两种方案：

# 4\.关于对积分提现规则的建议

之前一段时间我对酷狗音乐和汽水音乐那里的曲库业务进行了一些了解，酷狗那边的情况是每个月大概投入3万左右来奖励进行贡献的用户，汽水音乐是字节跳动旗下的新平台，我问了一下那边的运营，每个月奖励用户的投入最少在10万左右。

实际上，云小编平台对积分变现的呼声早已有之，我和社群的一些小编对该方面规则做出如下建议，希望你们能慎重考虑。（我们云小编真的很需要！！！）

考虑到积分量过于庞大会导致预算不足的问题，因而在变现汇率上做一些调整：目前积分商城兑换汇率大概在200:1，兑换现金的汇率可以上调至300:1左右。

同时，为规范行业健康性，可限制云小编等级15级以上者可兑换，需要后台强实名，限制18岁以上，并且**每月仅开放1至2次变现入口。**

倘若编辑部可投入的预算实在有限，也可进行限定数量，限定时间的提现，下列表格可以做一个参考，希望慎重考虑：



|入口开放时间|提现金额|所需积分|兑换数量|
|---|---|---|---|
|14点00分|80|25000|80|
|14点15分|100|30000|100|
|14点30分|200|60000|200|



统计了一下，达到兑换要求的小编总共550人左右，去掉那些不活跃的、账号已封禁的用户，底线300个名额刚好可以达到良性竞争的效果

# 5、Leader回复：

1. 第一个任务维护，我这可以尽快安排专人保障一下，目前的任务还不支持歌词和翻译的分发（有历史原因在），专辑艺人的资料缺失任务都是可以做的

2. 系统功能优化这部分，我们其实还有非常多想做的事情还在代办中，目前最大的瓶颈还是太缺开发人力了

3. 关于现金奖励，我们其实平均每个月用户奖励和兑换的投入要高于酷狗，但也没字节那么财大气粗，体现这个还涉及到一些系统支付之类的问题，做起来是非常复杂的，但我们可以考虑尝试一下针对头部用户单独开放专项的严选礼品卡投放，会比较接近现金奖励

4. 只靠运营向能解决的问题我这一定尽量帮大家满足，就是涉及到开发相关的内容，我不能给一个很明确的保证，但每个迭代我们都有相关需求提上去。

5. 感觉可以引入“纪检”角色，方便大家处理违规用户。回复内容：

    1. 我理解你们目前主要的需求还是去解决用户违规上传之类的，比如机翻。处理类似他人举报。不过机翻用户一般也不会去申诉自己的内容。可能可以更直接一点，纪检委用户的巡检结果或者举报，可以先限制权限，再收集证据，没问题再放出来（廉政公署模式：先抓人，再审问）

    2. 可以先用线下模式验证一下，可行性比较高的话再提功能需求。但现在真是排不上大需求，这个东西一听就是没一个迭代做不完

    3. 不过用户举报，就目前的已有的渠道看，会有大量无效反馈，很浪费资源，手机端的窗口一天能接到2000＋报错反馈，但无效内容占比很大，不仅浪费窗口资源，也浪费审核资源，毕竟你们时间也是有限的。功能其实并不难做，有需求的话还可以联动起来，甚至单开一个翻译投诉的类型。

# 6、一年后的改变

1. 任务在提出后立马就上线了，只不过是歌名翻译任务，其实这是和我的想法不同的，我并没打算恢复歌名翻译任务，而是想恢复歌词翻译任务。最后的效果挺不好的。

2. 没发现系统有什么优化的，缺开发缺一年了？

3. 现金奖励问题，目前维持积分制度不变，在汽水那边一年下来确实感觉到了一旦人变多，那么奖励资源就越匮乏。汽水的奖励成本早就超过十万，但依旧不够分配。更何况网易云积累了十年的用户资源。礼品卡还没有跟上头部用户，依旧是有限资源开放性抢夺。
