# 曲库优化方案

> *2025.03.24 · 作为用户向曲库团队提交的优化建议*

随着曲库规模的持续增长，现有审核与管理流程暴露出效率瓶颈与质量问题。以下为当时提交的系统性优化方案。

---

## 一、关于脚本上传问题的解决方案

### 用户类型与行为特征

| 类型 | 行为特征 |
|------|---------|
| 可疑用户 | 一分钟上传 > 3 首 / 设备频繁更换 / 集中在 0 点上传 |
| 新用户 / 低活跃用户 | 一分钟上传 ≤ 3 首 |

### 防控措施

- **防人机操作**：滑动验证码、图形文字识别认证
- **一分钟内上传超过 3 首歌** → 开启手机短信验证码验证
- **基础滑块验证**：滑块 + 短信验证
- **网安后台监控**异常行为（成本可能较大，甚至可能检测不到）

目前市面上已有成熟的 SDK 方案，例如腾讯云滑动拼图、字节鹊凿等，成本可控且落地快。

### 处罚机制

参考竞品汽水音乐（据说联合了法务），严厉一点：一旦有违规就直接封号（甚至不能听歌）、积分清零，每月公布违规用户昵称与 ID。

### 预期效果

- 拦截 80% 低阶脚本，误伤率 < 3%
- 减少人工审核量 15%

### 遗留内容与善后方案

一般被处理的用户所贡献的内容质量参差不齐，逐个排查易消耗审核成本。

---

## 二、关于报错机制与善后方案

### 手机端报错入口去留问题

- 收到无效反馈过多，浪费成本，直接砍掉或许是个更好的选择
- 但砍掉的话从普通用户层面就没有直接反馈渠道了
- **纪检委举报机制的设想**：为防止无效信息过多，只有云小编等级 Lv10 以上才可举报（可以给这部分用户单开入口）

### 用户层面问题

- 从用户层面来看，建议给举报内容板块做一个具体的表单，详细到一首歌哪一句错了，轴错了还是语法错了，给一个具体的规范
- **传送门**：用户哪知道什么叫传送门，只知道在评论区 @曲库。直接把报错入口换成传送门更好，收到的反馈会更详细。升级传送门接到报错通道后，用 DeepSeek 根据问题正确指引

### 去年提过的纪检委方案

推进尚不明确。其实都是针对个别用户，但对他们而言一个号没了再起一个就行。或许提交时加个验证码就是成本最低的方案，但也容易牺牲一小部分普通用户。

---

## 三、引入 AI 的方案

### 1. 用户侧 AI 助手

- 用户层面不知道的太多，怎么让用户正确解决也是个问题。虽然已经有云小编总则了，但没人会翻阅那么多文本内容
- 考虑问题的多样性，给 AI 喂数据来一对一回答或许更好

### 2. 云小编 AI 助手

可借鉴美团"小象"工单系统，预设 100+ 常见问题模板。在云小编中心也可以放一个 AI 来处理初级问题，可以先在部分云小编群体进行灰度测试。

## 四、好奇咱们能不能搞一个盖章

就是那种实名制的官方实践证明。

### 参考案例

| 平台 | 认证体系 |
|------|---------|
| 云音乐社区 | 面向校园的实践证明 |
| 字节 | 面向类似云小编的群体，"校园创作营"认证体系，可对接高校学分认定 |
| 拉钩 × B 站 | "创作力认证" |
| 猎聘 × 腾讯 | "技术大咖认证" |
| 脉脉 × 网易云 | 音乐人认证 |

实践中编码可对接招聘软件的"大厂人才认证系统"。
