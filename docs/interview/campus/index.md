## 疯狂星期四 · 歌词翻译激励活动

> 被问到次数最多的项目，几乎每家必问。

当时曲库对歌词翻译 UGC 的运营遇到小问题：非节假日时段用户上传积极性不强，以往活动参与门槛太高。活动目标：既能让大家轻松参与又能拿到实在的奖励。

**我的角色**：拟定主题 → 撰写宣推文案 → 搭建 H5 → 配置玩法 → 选取商品 → 估算成本 → 上线后实时观察舆情 → 收集用户信息发放奖励 → 复盘归因异常数据。最终效果对比去年同期和非活动期都有很好的增长。

**吸引用户的策略**：

| 维度 | 策略 |
|------|------|
| 宣推主题 | 采用"疯狂星期四"大众化主题，用户接受度高 |
| 活动玩法 | 双倍积分 + 双抽奖池，阶梯引导（5 道具 = 高概率） |
| 用户心理 | 利用"赌一赌"心态 + 积攒进阶的成就感 |

**成本估算方法**：正式上线前在用户社群小范围试点（仅告知周四有翻倍活动），以第一期数据为基准估算第二期收益。

**异常数据归因**：
1. 发现一个异常用户在第一期单人上传四五百首歌词后完全消失 → 改进策略：加强与这类用户的联系，争取触达
2. 第一期恰逢欧美区新歌发布热潮，第二期无新动作 → 活动策划要兼顾关键时间节点

## 汽水音乐 · 奖项 SOP 搭建

> 昆仑、东西世界、荔枝均深入问到。

核心思路：**先定目标 → 拆解场景 → 制定规则 → 补充细节**

当时问题是人工维护奖项信息导致来源不统一、效率低下。核心目标：让所有维护人员按统一逻辑获取、补充数据，确保准确度且可追溯。

**拆解为五个固定流程**：获奖周期 → 奖项类型 → 实体类型 → 获奖类型 → 关联曲库内容。每个步骤给出明确规则（如周期补近十届、实体类型看奖项名称后缀、版本不一致要标注）。

**细节补充**：奖项已停办的情况 —— 从最后一届开始补充十届，不到十届就补到第一届。

## 竞品分析 · 汽水音乐 vs 网易云音乐

- **用户群体差异**：网易以内容驱动型为主（我喜欢这首歌所以我想贡献），汽水以价值驱动型为主（你们给了较高激励所以我愿意贡献）
- **审核标准**：网易相对宽松，只要不是直观离谱错误都会通过，依靠纠错体系弥补（QQ、传送门、站内客服、百科入口 → 最终同一个人处理）；汽水审核严格，需提交逐字滚动
- **汽水可借鉴之处**：独家任务领取制（24 小时内完成）、专业审核纠错能力

---

## 网易云音乐的挑战与应对

当前处于"前有腾讯系（Q 音、酷狗）后有汽水"的格局。云音乐的优势是**高价值深度消费用户**——这是腾讯和汽水永远打不过来的三八线。

| 策略方向 | 要点 |
|----------|------|
| 版权策略 | 头部筑基，小众破局 |
| 用户资产管理 | 升级用户资产管理体系 |
| 音乐发现 | 构建专业化音乐发现系统 |
| 算法与人工 | 平衡算法与人工推荐 |

## 多元文化探索 · 账号运营项目

这是一个构建音乐垂类内容生态的项目，负责三大核心品类（资讯、推荐、科普）、11 个细分方向的全流程内容运营。

| 维度 | 详情 |
|------|------|
| 量化指标 | 日均发布 ≥ 6 条，三大类各 ≥ 2 条，绝不允许超时 |
| 发布规范 | "发布四件套"：精准话题、10 字以上文案、3 张以上无站外水印配图、站内版权歌曲 |
| 执行保障 | "发布前双检 SOP"：品类达标检查 + 物料合规检查，零违规零遗漏 |
| 最终成果 | 发布总量排名项目前 5%，单条互动量进入全站前 1% |
