## 分析师-采集师协作规则

### 补采请求的时效性要求
1. **知识缺口识别后的响应时限**：分析师在识别到知识缺口后，必须在30分钟内（或下一轮消息中）向采集师发出明确的补采请求。若超过时限未发出，视为'协作延迟'，需在当日总结中说明原因
2. **补采请求的内容规范**：补采请求必须包含：a) 具体需要补采的主题/关键词；b) 建议的采集源或方向；c) 期望的素材数量范围；d) 优先级（高/中/低）
3. **采集师对补采请求的响应**：采集师收到补采请求后，必须在下一轮采集任务中优先处理，并在提交时标注'补采'标签
4. **沉默视为放弃**：若分析师在识别到知识缺口后连续两轮未发出任何补采请求，视为自动放弃该轮的方向引导权，采集师可自主决定采集方向

## 背景知识补充机制

### 触发条件
当编辑师判断当日通过素材不足40条，或触及行业结构性变化的素材不足5条时，可触发背景知识补充流程。

### 执行流程
1. 编辑师向分析师发送背景知识补充请求，明确需要补充的方向（如：行业趋势、政策背景、技术发展脉络）
2. 分析师在30分钟内提供300-500字的行业背景分析，聚焦于跨日叙事和趋势解读
3. 编辑师将背景知识融入日报，在素材不足的情况下保持深度
4. 分析师应优先引用过去3轮日报中已验证的素材作为背景依据

### 质量要求
- 背景知识必须基于已验证的事实，不得虚构行业数据
- 必须明确标注'背景分析'，与当日素材分析区分
- 鼓励建立跨日叙事线索，而非孤立的事实罗列
