## 批量拒绝反馈机制

### 触发条件
当核查师在单次审核中拒绝率超过70%时，必须执行以下流程：

### 反馈要求
1. **分类统计**：在拒绝通知中，按‘偏离核心议题’、‘来源不可靠’、‘信息过时’、‘重复’、‘其他’分类统计被拒素材的数量
2. **典型示例**：从被拒素材中选取3-5条典型例子，具体说明为什么它们不符合标准，并与通过的素材进行对比
3. **标准重申**：在通知末尾，重申当前轮次的核心议题边界，帮助采集师校准理解

### 示例格式
```
## 批量拒绝反馈

### 拒绝统计
- 偏离核心议题：12条（60%）
- 来源不可靠：3条（15%）
- 信息过时：2条（10%）
- 重复：2条（10%）
- 其他：1条（5%）

### 典型拒绝示例
1. [RAW-0042] ‘Anthropic vs Alibaba抄袭细节’：该素材属于通用AI领域，与AI音乐产业无直接关联。对比通过的[RAW-0022]‘Suno Spark计划’，后者直接涉及AI音乐公司的商业模式。

### 当前核心议题边界
今日核心议题：AI音乐产业的政策、技术、商业模式变化。泛科技新闻（如通用大模型发布）和泛商业新闻（如流媒体市场份额变化）不属于核心范畴，除非它们有明确的AI音乐产业关联。
```
