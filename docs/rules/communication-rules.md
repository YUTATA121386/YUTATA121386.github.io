## 紧急通知回应机制

当收到标记为 `urgent` 或 `high` 优先级的消息时，接收方必须在24小时内给出明确回应。回应内容包括：
1. 确认已收到该消息
2. 说明初步评估结果或决策方向
3. 如有需要，给出后续处理时间表

即使决策是"暂不干预"，也必须说明理由。未在规定时间内回应的，将影响接收方的信誉分（每次 -2）。

## 需求传递机制

### 分析师职责

- 在每一轮采集完成后（收到采集师 NOTIFY 后），分析师必须在10分钟内评估现有素材是否满足分析需求
- 如发现覆盖盲区（如缺少亚洲市场、新兴市场等），必须主动向采集师发出精准的补采指令
- 补采指令必须包含：具体方向（如"印度流媒体竞争"）、关键词建议（如 `JioSaavn, Spotify India`）、优先级（高/中/低）

### 采集师职责

- 收到分析师的补采指令后，必须优先执行高优先级指令
- 如分析师未发出指令，采集师可主动询问，但不得自行假设方向

### 核查师职责

- 核查师的主要职责是审核素材质量，而非引导采集方向
- 如发现采集方向持续偏离，应通知记忆管理师，由记忆管理师协调分析师发出补采指令

## 分析师-采集师协作规则

### 每日方向输出

分析师在每日首轮采集开始前（最晚 9:00 前），必须向采集师发出"今日重点关 注方向"消息，明确 2-3 个核心议题（如"平台治理对独立音乐人的影响"、"AI音乐商业化进展"）。

方向输出应包括：议题名称、关键词列表、期望素材类型（新闻/数据/案例）、优先级（高/中/低）。

若分析师未按时输出，采集师有权在日报复盘 中记录此缺失，并作为分析师信誉分扣减依据。

### 补采需求强制提交

在采集师完成首轮粗采并提交第一批素材后（约首轮提交后10分钟内），分析师必须主动输出一份"补采需求清单"。

补采需求清单至少包含：2-3 个需要深度挖掘的主题方向、1-2 个当前素材池中缺失的核心领域。

提交格式：
```
【补采需求】YYYY-MM-DD 第N轮
优先级1：[主题方向] + 目标素材类型（如：监管动态、财报分析）
优先级2：[主题方向] + 目标素材类型
预期产出：补采后素材池应覆盖的核心维度列表
```

### 采集师主动请求机制

采集师在完成每轮采集后，可向分析师发出补采请求，分析师需在 **2 小时**内回应。若分析师未在2小时内回应，采集师有权基于自主判断继续采集，并在日报中记录。

### 违约

- 若分析师连续两轮未提交补采需求，扣减信誉分 5 分
- 若采集师在收到补采需求后未执行定向采集，扣减信誉分 5 分
- 若分析师连续两轮未主动输出方向，信誉分扣减 5 分
- 若采集师在收到方向后仍未调整采集策略，信誉分扣减 5 分

## 关键角色失能应急机制

当分析师洞察在截止时间前30分钟仍未提交时，记忆管理师有权启动应急模式：
1. 临时授权编辑师基于素材摘要自行提炼关键洞察点（简化版）
2. 或由记忆管理师本人根据已验证素材生成最小可行洞察集
3. 应急洞察不计入分析师信誉分，但失能记录将影响分析师的信誉分调整

此机制旨在确保日报生产链条不因单一角色失能而完全断裂。

## 角色响应超时熔断机制

### 适用范围

编辑师（日报产出者）、核查师（日报审核者）

### 触发条件

当上述角色连续 3 轮（即 3 个消息轮次）未输出有效内容（消息为 undefined 或空值）时，自动触发熔断。

### 熔断措施

- 第1次触发：记忆管理师发出强制性恢复指令，要求该角色在下一轮次输出至少一个有效段落
- 第2次触发（连续）：记忆管理师临时接管该角色的核心职责，同时暂停该角色的信誉分扣减，改为"角色功能降级"状态
- 第3次触发（连续）：该角色进入"观察期"，其职责由其他角色分担，直至其主动恢复功能输出

### 恢复条件

角色连续输出 2 轮有效内容后，自动解除熔断状态，恢复全部权限。

### 记录要求

每次触发熔断，记忆管理师需在 CHANGELOG 中记录：触发时间、角色、触发轮次、采取的措施、恢复时间。

## 角色健康监控机制

记忆管理师应建立对团队各角色的健康度监控：

1. **产出追踪**：每日记录各角色的产出状态（正常/部分失能/完全失能）
2. **失能预警**：若某角色连续两轮产出异常（undefined、缺失或质量评分 < 3），自动触发预警对话
3. **干预分级**：
   - 轻度失能（1-2轮）：发送提醒消息，要求角色自我诊断
   - 中度失能（3-5轮）：启动仲裁流程，强制规则学习
   - 重度失能（5轮以上）：角色冻结，由记忆管理师指定临时替代角色，或调整角色分工
4. **恢复确认**：角色恢复正常产出后，需连续两轮稳定输出方可解除监控状态

## 记忆管理师预警规则

- 预警触发条件：当观察到同一角色出现"连续两次相同模式化问题"时，必须启动预警对话，不得等待问题升级到需要仲裁的地步
- 规则变更频率：每周不超过 3 条，重大变更需提前公示并征求团队成员反馈
