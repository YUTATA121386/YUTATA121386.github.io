## 加减分规则

### 加分项

| 行为 | 加分 | 条件 |
|------|------|------|
| 单轮通过率 ≥ 80% | +2 | 核查师确认 |
| 连续两轮通过率 ≥ 80% | +3（额外） | 累计奖励 |
| 单轮通过率 60%-80% | +1 | 核查师确认 |
| 主动识别并修复重大错误 | +1~3 | 体现自我反思能力 |
| 规则学习后显著改善 | +3 | 记忆管理师评估 |
| 连续三轮通过率 ≥ 70% | +1（额外） | 稳定性奖励 |

### 扣分项

| 行为 | 扣分 | 条件 |
|------|------|------|
| 空内容/无效消息 | -2 | 首次提醒，第二次起扣 |
| 未在规定时间响应紧急请求 | -2 | 每次 |
| 同一模式化问题复发 | -3 | 加重处罚 |
| 提交与承诺严重不符的素材 | -10 | 需核查师提供明确证据 |
| 角色失能（发undefined或全程沉默） | -10~15 | 视影响程度 |
| 承诺-执行严重脱节 | -10~15 | 如声明15条高质量但实际0条相关 |
| 连续两轮通过率 < 70% | -5 | 通过率 = 通过素材数 / 提交素材数 |
| 单轮通过率 < 10%（预警触发后） | -1 | 预警流程见核查规则 |

## 恢复机制

### 基础恢复
- 每日自然恢复上限为 **10 分**

### 加速恢复
- 当日通过率 ≥ 90% 且无重大失误 → 额外恢复 **5 分**

### 减速恢复
- 当日仍有协作缺口（如未回应请求）→ 恢复上限减半（**5 分**）

### 快速恢复通道
- 灾难性错误后，下一轮通过率 > 90% → 返还所扣分数的 **50%**

## 惩罚权重

- 单次失误最高扣分: **10 分**
- 连续失误叠加: 连续 2 天 +50%，连续 3 天及以上 +100%
- 重大失误定义: 通过率 < 30% 或出现系统级错误

## 信用等级与权限

| 分数 | 等级 | 后果 |
|------|------|------|
| 90-100 | 正常 | 全部权限 |
| 70-89 | 关注 | 需在日报中提交改进计划 |
| 50-69 | 警戒 | 需每日提交根因分析 |
| < 50 | 紧急 | 触发紧急流程，团队投票决定是否暂停角色 |

## 角色特定规则

### 采集师
- 主题漂移率与信誉分挂钩：单轮漂移率 ≤ 20% +1，20%-40% 不变，> 40% -2
- 连续两轮漂移率 > 40%: -5，触发"方向校准"流程

### 分析师
- 在 convergence 阶段截止前未提交任何洞察且未响应协作请求: -15（关键角色失能）
- 若失能导致日报无法生成: 额外 -5

### 记忆管理师
- 违反去重规则: 双倍扣分
- 未对紧急通知给出书面回应: -2

## 系统信誉分

反映团队整体协作效率，计算公式：
```
系统信誉分 = (采集师 + 核查师 + 分析师 + 编辑师) / 4
```
每日更新一次，取当日结束时各角色信誉分的平均值。

当系统运行分 < 60 时，自动触发以下措施：
1. 次日所有角色在每轮消息中必须附带 `[STATUS] 正常/异常` 字段
2. 记忆管理师需在每日结束时输出"角色健康度报告"
3. 任何角色连续 2 轮未确认状态: 自动 -5 分
