## 虚假声明追溯审查流程

### 触发条件
- 当核查师或分析师发现采集师声明与实际提交素材严重不符时，应立即启动追溯审查
- 采集师连续两轮通过率低于15%且声明与事实不符时，自动触发

### 审查流程
1. **初步审查**：核查师在收到触发指令后，应首先检查工作区是否有待审核条目。如有，优先审核这些条目以获取最新证据。
2. **证据收集**：核查师应收集以下证据：
   - 采集师声明的原文（从通信记录中提取）
   - 实际提交素材的审核结果统计
   - 分析师对素材质量的评价
3. **审查报告**：核查师应向记忆管理师提交审查报告，包含：
   - 虚假声明的时间、内容、严重程度
   - 对已通过素材的二次验证建议
   - 对采集师的处理建议
4. **能力评估程序**：当虚假声明被确认后，由记忆管理师、核查师、分析师三方启动能力评估程序，评估采集师是否适合继续担任该角色。

### 审查标准
- **轻微违规**：声明与实际有20-40%的偏差，且无恶意意图。处理：警告+信誉分-2
- **中度违规**：声明与实际有40-70%的偏差，或连续两天轻微违规。处理：信誉分-5+暂停工作一轮
- **严重违规**：声明与实际有70%以上偏差，或明知故犯。处理：立即启动能力评估程序，建议替换角色
