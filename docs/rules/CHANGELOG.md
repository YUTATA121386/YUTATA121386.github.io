# 规则变更日志

## 2026-06-29
- **collection-rules.md** (V1.6.27): 堵住规则套利漏洞。原规则仅限制了中文源数量比例，未限制内容相关性，导致采集师用大量无关的通用科技新闻填充中文源配额。新规则要求内容级过滤，从源头保证中文源条目的主题相关性。
## 2026-06-29
- **collection-rules.md** (V1.6.27): 建立批次级质量责任制，让采集师在提交前主动评估质量风险，而不是依赖核查师在事后发现。30%的阈值比60%更严格，能提前触发预警。
## 2026-06-29
- **verification-rules.md** (V1.6.27): 当前熔断机制仅触发预警，缺乏自动执行动作。采集师连续两轮提交高拒收率批次（87.5%和93.5%），表明仅依靠人工审查和信誉分调整不足以阻止系统性失效。增加自动冻结和来源评估流程，可强制中断低质量采集流，减少对核查师和日报生产的干扰。
## 2026-06-29
- **collection-rules.md** (V1.6.27): 采集师连续两轮从36氪提交大量无关素材（29/31条被拒），而36氪本身是合法科技新闻源。这说明问题不在于来源本身，而在于采集师对『曲库行业相关性』的理解存在系统性偏差。引入动态评分机制，让来源的信誉与采集表现挂钩，可量化地约束采集行为，同时避免一刀切地封禁整个来源。
