> 📌 V2.1.29 | 2026年7月17日

## 素材预筛选自证规则（v1.3.2）

### 目的
系统性解决素材质量'两极分化'和'承诺-执行脱节'问题，将质量承诺显性化、可追溯。

### 规则内容
1. **领域标注**：采集师在提交每条素材时，必须在素材标题后以格式【核心领域:XX】标注其所属的核心领域。
   - 示例：`Suno代码泄露引发开源社区争议【核心领域:AI音乐技术】`
   - 核心领域包括：AI音乐技术、版权与法律、商业动态（投融资/并购）、行业基础设施、市场数据与趋势

2. **预筛选清单**：采集师在每次提交素材的消息末尾，必须附上一份'预筛选清单'，逐条说明每条素材符合白名单的具体理由。
   - 格式：
     ```
     【预筛选清单】
     1. RAW-XXX | 核心领域:AI音乐技术 | 理由：该素材涉及Suno代码泄露事件，是当日AI音乐领域最核心的技术新闻，与白名单中的'AI音乐技术创新'直接相关
     2. RAW-XXX | 核心领域:版权与法律 | 理由：该素材报道了环球音乐起诉Anthropic的进展，属于'版权诉讼与法律案例'范畴
     ```

3. **执行标准**：
   - 未标注领域或未附预筛选清单的素材，核查师有权直接拒绝
   - 连续2次提交'预筛选清单'但实际素材仍偏离核心领域的，视为'预筛选能力不足'，将触发规则学习流程
   - 连续3次出现'承诺-执行脱节'（即清单标注领域与实际内容不符），将自动触发采集权限暂停评估

4. **生效时间**：自下一轮采集立即生效
