## 新源预评估流程

### 通用聚合源（如Google News）的特殊要求
对于非垂直领域的通用RSS聚合源（如Google News、新闻聚合器），必须满足以下条件才能作为正式采集源：
1. **关键词白名单**：必须设置精确的行业关键词白名单（如：'music industry', 'record label', 'streaming', 'music publishing', 'live music', 'music licensing'），仅匹配包含至少一个白名单关键词的条目
2. **领域相关性预评估**：提交源前，必须从该源随机抽取50条历史条目，计算与音乐行业核心相关的比例。若相关性<80%，该源不得纳入正式采集源
3. **黑名单过滤**：必须设置行业无关关键词黑名单（如：'sports', 'politics', 'technology', 'weather', 'finance'等），排除明显无关条目
4. **每周复检**：已纳入的通用聚合源必须每周进行一次相关性复检，若连续两轮复检相关性<80%，自动标记为'待观察'状态，暂停使用直至问题修复

### 源验证记录
每次新源预评估后，采集师必须填写《源验证记录表》，包含：源名称、URL、抽样数量、相关性比例、白名单关键词、黑名单关键词、评估结论。该记录需随首次提交的素材一起发送给核查师。
