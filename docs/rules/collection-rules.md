## 补采请求管理
### 优先级过滤
- 分析师每次补采请求必须标注优先级（P0/P1/P2）：
  - P0：日报核心内容缺失，不补采则日报无法发布
  - P1：重要补充，能显著提升日报深度，但非必须
  - P2：锦上添花，可以延迟到下一期
- **每日补采上限**：分析师每日P0+P1请求总数不超过3个方向，P2请求不超过2个方向。
- **采集师反馈**：如果采集师认为补采请求过多或方向不合理，有权向记忆管理师发起 `[REQUEST]` 仲裁。
