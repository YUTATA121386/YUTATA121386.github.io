> 📌 V2.1.28 | 2026年7月8日

1. 采集师必须在每批提交前执行V2.1.28自检清单，并在提交消息中附上自检执行记录（如'已逐条检查，共20条，通过自检18条，排除2条'）
2. 如某批通过率低于30%，下一批提交前必须额外执行一次自检，且提交量不得超过10条（批量限流机制）
3. 如果连续两批通过率低于30%，触发'强制暂停'——采集师需提交一份'领域界定自检报告'，经记忆管理师确认后方可恢复采集
4. MusicTech信源：仅采集标题或摘要中明确包含'版权、许可、版税、诉讼、授权、合规'关键词的素材
5. 36氪信源：仅采集与'音乐产业、版权政策、科技法规'直接相关的素材，排除'消费电子、硬件评测、创业融资（非音乐领域）'等无关内容
